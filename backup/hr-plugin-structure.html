플러그인 구조 해설

Cowork(에이전트형 데스크톱 앱)를 위해 설계되고 Claude Code에서도 동작하는 피플 오퍼레이션스 플러그인의 전체 구성을 한눈에 정리한 문서입니다. 채용·온보딩·성과관리·정책 안내·보상 분석을 다룹니다.

name · human-resources version · 1.2.0 author · Anthropic primary · Cowork + Claude Code
01 / OVERVIEW

플러그인이란 무엇인가

플러그인은 범용 Claude를 특정 직무의 전문가로 바꿉니다. HR 플러그인은 피플팀이 실제로 일하는 방식 — 워크플로우, 용어, 산출물 형식 — 을 반영하도록 실무자와 함께 설계되었습니다. 통합 없이 단독(Standalone)으로도 동작하며, 사내 HRIS·ATS·지식베이스를 연결하면 능력이 증폭(Supercharged)됩니다.

Commands — 슬래시로 직접 호출하는 명시적 워크플로우 Skills — 상황에 맞게 자동 호출되는 도메인 지식 Settings / Connectors — 회사 맞춤 설정과 데이터 연결
02 / ARCHITECTURE

세 개의 층으로 이뤄진 구조

사용자의 요청은 두 갈래로 처리됩니다. 명령을 명시적으로 입력하면 Commands가, 자연스러운 질문을 던지면 관련 Skills가 자동으로 깨어납니다. 두 층 모두 Settings·Connectors 층에서 회사 정보와 실데이터를 끌어옵니다.

User / 피플팀 “신입 온보딩 계획 짜줘”  ·  “우리 육아휴직 정책 뭐야?”  ·  /comp-analysis

05 / STANDALONE → SUPERCHARGED

단독으로도, 연결하면 더 강력하게

모든 커맨드와 스킬은 통합 없이도 작동합니다. 정보를 직접 입력하면 단독으로 돌고, 사내 시스템을 MCP 커넥터로 연결하면 실데이터로 자동 채워집니다.

07 / SETTINGS

회사 맞춤 설정 — settings.local.json

이 파일 하나로 회사 정보를 주입하면 산출물이 사내 표준에 맞게 나옵니다. 파일이 없으면 플러그인이 대화형으로 직접 물어보므로, JSON을 다루지 않아도 채팅으로 입력할 수 있습니다.

{ "company": "Your Company", "headquarters": "City, State", "employeeCount": 500, "benefits": { "healthInsurance": "Provider Name", "pto": "Unlimited / X days", "parentalLeave": "X weeks" }, "compensation": { "currency": "USD", "equityType": "RSU / Options", "vestingSchedule": "4 years, 1 year cliff" } }
저장 위치  ·  Cowork에서는 폴더 피커로 공유한 임의의 폴더에 저장하면 플러그인이 자동으로 찾습니다. Claude Code에서는 human-resources/.claude/settings.local.json 경로에 둡니다.
한국 현지화 주의  ·  공식 플러그인은 관할권 오버레이가 없어 미국식 복리후생·보상 가정이 기본값입니다. 삼성SDS 적용 시 근로기준법·연차 규정·취업규칙·개인정보보호법에 맞춰 정책 지식베이스와 설정을 새로 채워야 하며, 정책·노무 Q&A는 "직원 자가응답"보다 "HR 담당자 초안 보조" 모드로 시작하는 것이 안전합니다.
08 / INSTALLATION

설치

엔터프라이즈 관리자는 Cowork의 Customize 메뉴에서 사설 마켓플레이스를 만들고 이 플러그인을 가져와 사내 템플릿으로 복제할 수 있습니다. Claude Code에서는 아래 명령으로 추가합니다.

$ claude plugins add knowledge-work-plugins/human-resources

본 문서는 공식 anthropics/knowledge-work-plugins 저장소의 HR 플러그인 README 및 manifest(v1.2.0) 구성을 구조화·시각화한 자료입니다.
