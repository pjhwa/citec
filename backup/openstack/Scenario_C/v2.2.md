# CSAP 9.1.4-③ 하이퍼바이저 백신 요건 대응 논리 v2.2 (SCPv2 Ubuntu/KVM)

> **v2.2 변경 사항**:
> 1. **현재 환경 상태 반영**: MAC(sVirt/AppArmor)이 Ubuntu + libvirt에 **기본 탑재**되어 있으나 **현재 disable 상태**임. 따라서 "신규 도입"이 아닌 "**활성화·강화**" 관점으로 전환.
> 2. **타임라인 대폭 수정**: 본심사 전까지는 **Test 환경에서만** 검증 완료. 본심사 통과 후 **Dev 환경부터 본격 적용**하는 현실적·안전한 일정으로 변경.

## 3. 주장 2: 보완통제(MAC)의 동등성·우월성

### 3-1. MAC의 작동 원리 — 프로세스 단위 격리 (수정)

**중요 사실**: Ubuntu 22.04/24.04 + libvirt 환경에서는 **AppArmor가 기본 설치**되어 있으며, libvirt는 기본적으로 sVirt를 지원한다.  
다만 **현재 우리 환경에서는 `security_driver = "none"` 또는 미설정으로 disable 상태**이다.

따라서 본 대응은 **"신규 설치"가 아닌 "이미 존재하는 MAC을 enable하고 프로파일을 강화"**하는 작업이다.

| 차원 | 전통 AV | **MAC (sVirt/AppArmor)** |
|------|---------|---------------------------|
| 탐지 방식 | 시그니처 기반 (사후) | **정책 기반 (사전)** |
| 작동 계층 | 사용자 공간 | **커널 공간** |
| Zero-day 대응 | 패턴 업데이트 후 탐지 | **정책 위반은 무조건 차단** |
| VM-VM 격리 | 불가능 | **VM 프로세스별 고유 프로파일** |
| 성능 오버헤드 | 실시간 스캔 시 높음 | **1-3% 미만** |
| 현재 우리 환경 | - | **기본 탑재 + disable 상태** → 활성화 필요 |

## 6. 수정된 3개월 검증·적용 로드맵 (본심사 안전 최우선)

> **현재 상태**: MAC(sVirt/AppArmor)이 Ubuntu + libvirt에 **기본 탑재**되어 있으나 **disable 상태**이다.  
> 따라서 3개월 로드맵은 "**신규 도입**"이 아니라 "**활성화·프로파일 강화·운영화**"이다.

### 6-1. 전체 타임라인 (v2.2)

| Phase | 기간 | 대상 환경 | 주요 목표 | 심사 대응 |
|-------|------|-----------|----------|----------|
| **Phase 0** | 4/20 ~ 4/26 | 전 환경 | 프로파일 템플릿 작성 + baseline 수집 | - |
| **Phase 1** | **4/27 ~ 5/17** (3주) | **Test 환경만** | Complain → Enforce 전환 + 3주 안정성 검증 | **사전점검(~4/29)**: Phase 1 진행 중<br>**본심사(5/18~22)**: Phase 1 완료 증적 제출 |
| **Phase 2** | 5/23 ~ 6/13 (3주) | **Dev 환경** | Complain → Enforce + 검증 | 본심사 **통과 후** 본격 시작 |
| **Phase 3-1** | 6/14 ~ 6/28 (2주) | 운영 Canary (3~5노드) | Complain → Enforce | - |
| **Phase 3-2** | 6/29 ~ 7/12 (2주) | 운영 환경 50% | Enforce 확대 | - |
| **Phase 3-3** | 7/13 ~ 7/20 (1주) | 운영 환경 전 노드 | Enforce 100% 완료 | 3개월 내 목표 달성 |

### 6-2. 심사 시점 대응 상태 (v2.2)

- **사전점검(~4/29)**: Phase 1 진행 중 — "Test 환경에서 MAC 활성화 검증 중" 증적 제시
- **본심사(5/18~22)**: Phase 1 완료 시점 — "**Test 환경 3주 검증 완료 + 본심사 통과 후 Dev 환경부터 본격 MAC 활성화 계획**" 증적 제시
- **본심사 이후**: Phase 2(Dev) → Phase 3(운영) 순차 적용

### 6-3. 각 Phase의 검증 기준 (v2.2)

| Phase | 합격 기준 |
|-------|----------|
| Phase 1 | **Test 환경**에서 VM 부팅/마이그레이션 100% 성공, DENIED 로그 0건, enforce 모드 안정 운영 2주 이상 |
| Phase 2 | **Dev 환경**에서 성능 저하 < 3%, DENIED 로그 0건 |
| Phase 3 | 운영 환경 Canary 2주 무장애 → 전 노드 Enforce |
