## 2. E-01: AppArmor 기본 상태

### 2-1. 수집 명령

```bash
# 전체 AppArmor 상태
sudo aa-status

# JSON 형식 (자동화용)
sudo aa-status --json > /var/log/apparmor/aa-status-$(hostname)-$(date +%Y%m%d).json

# 커널 지원 확인
cat /sys/kernel/security/apparmor/profiles | wc -l
```

### 2-2. 예상 출력 (운영 상태)

```
apparmor module is loaded.
87 profiles are loaded.
85 profiles are in enforce mode.
   /usr/sbin/libvirtd
   /usr/lib/libvirt/virt-aa-helper
   libvirt-<uuid-1>
   libvirt-<uuid-2>
   ...
2 profiles are in complain mode.
0 profiles are unconfined but have a profile defined.
```

### 2-3. 합격 기준

- [ ] `apparmor module is loaded` 확인
- [ ] `libvirtd`, `virt-aa-helper` enforce 모드
- [ ] VM 개수 = `libvirt-<uuid>` 프로파일 개수
- [ ] unconfined 프로파일 0개

## 3. E-02: libvirt sVirt 활성 확인

### 3-1. 수집 명령

```bash
# sVirt 활성 확인
virsh capabilities | grep -A3 '<secmodel>'

# libvirt qemu.conf 설정 확인
sudo grep -E '^security_driver' /etc/libvirt/qemu.conf

# OpenStack Helm 환경: libvirt Pod 내부에서 확인
kubectl exec -n openstack $(kubectl get pod -n openstack -l application=libvirt -o jsonpath='{.items[0].metadata.name}') -- virsh capabilities | grep -A3 '<secmodel>'
```

### 3-2. 예상 출력

```xml
<secmodel>
  <model>apparmor</model>
  <doi>0</doi>
</secmodel>
```

### 3-3. 합격 기준

- [ ] `<model>apparmor</model>` 확인
- [ ] `security_driver = "apparmor"` 명시적 설정

## 4. E-03: VM별 AppArmor 프로파일 자동 생성

### 4-1. 수집 명령

```bash
# 동적 생성된 VM별 프로파일 목록
ls -la /etc/apparmor.d/libvirt/libvirt-* | head -20

# 프로파일 내용 예시 (대표 1개)
sudo cat /etc/apparmor.d/libvirt/libvirt-$(virsh list --name | head -1 | xargs virsh domuuid)

# 프로파일 수 집계
ls /etc/apparmor.d/libvirt/libvirt-*.files 2>/dev/null | wc -l
```

### 4-2. 예상 파일 쌍

- `/etc/apparmor.d/libvirt/libvirt-<UUID>` — 프로파일 본문
- `/etc/apparmor.d/libvirt/libvirt-<UUID>.files` — VM별 허용 파일 목록 (디스크, pid, 로그)

### 4-3. 합격 기준

- [ ] 실행 중 VM 수 == 프로파일 쌍 수
- [ ] 프로파일에 `/var/lib/nova/instances/<VM_UUID>/` 접근 허용 명시

```bash
ps -eZ | grep qemu-system | awk '{print $1, $NF}'
```

## 7. E-06~07: AIDE FIM 증적

### 7-1. AIDE 설치 및 베이스라인

```bash
# 설치 (Ubuntu)
sudo apt install aide aide-common -y

# 설정 파일
sudo vim /etc/aide/aide.conf
# (감시 대상/제외 경로 정의)

# 베이스라인 생성
sudo aideinit
sudo mv /var/lib/aide/aide.db.new /var/lib/aide/aide.db
sudo cp /var/lib/aide/aide.db /var/lib/aide/aide.db.$(date +%Y%m%d).bak
```

### 7-2. 일일 검사 스크립트 (자동화용)

```bash
#!/bin/bash
# /usr/local/bin/aide-daily-check.sh

LOG_FILE="/var/log/aide/aide-check-$(date +%Y%m%d).log"
mkdir -p /var/log/aide

/usr/bin/aide --check > "$LOG_FILE" 2>&1
RESULT=$?

if [ $RESULT -ne 0 ]; then
  # 변경 탐지 → Alert
  logger -p local0.warning "AIDE: changes detected, see $LOG_FILE"
  # SIEM 전송
  curl -X POST https://siem.internal/api/alert \
    -d "@$LOG_FILE" -H "Content-Type: text/plain"
fi

# 감사용 증적 영구 보관
cp "$LOG_FILE" /mnt/audit-archive/aide/
```

### 7-3. Cron 설정


## 8. E-08: 호스트 OS 패키지 인벤토리

### 8-1. 수집 명령

```bash
# 전체 설치 패키지 목록
dpkg --get-selections > /var/log/dpkg-inventory-$(hostname)-$(date +%Y%m%d).txt

# 패키지 수 집계
dpkg -l | grep '^ii' | wc -l

# 취약 도구 미설치 확인 (음성 증적)
for pkg in gcc make build-essential telnet ftp nc netcat tcpdump; do
  dpkg -l $pkg 2>/dev/null && echo "WARN: $pkg 설치됨" || echo "OK: $pkg 미설치"
done
```

### 8-2. 일일 diff 자동화

```bash
#!/bin/bash
# /usr/local/bin/dpkg-diff-check.sh

CURRENT=/tmp/dpkg-current.txt
BASELINE=/var/lib/dpkg-baseline.txt
dpkg --get-selections > "$CURRENT"

if ! diff -q "$BASELINE" "$CURRENT" > /dev/null; then
  diff "$BASELINE" "$CURRENT" | logger -p local0.warning -t dpkg-diff
  diff "$BASELINE" "$CURRENT" > /var/log/dpkg-diff/dpkg-diff-$(date +%Y%m%d_%H%M).log
fi
```

## 13. 증적 수집 자동화 스크립트 (통합)

```bash
#!/bin/bash
# /usr/local/bin/csap-evidence-collect.sh
# 용도: CSAP 심사 증적 일괄 수집

set -e
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
HOST=$(hostname)
OUTPUT_DIR="/var/log/csap-evidence/${HOST}_${TIMESTAMP}"
mkdir -p "$OUTPUT_DIR"

echo "[1/10] AppArmor 상태..."
aa-status --json > "$OUTPUT_DIR/E01_aa-status.json" 2>&1

echo "[2/10] libvirt sVirt 확인..."
virsh capabilities > "$OUTPUT_DIR/E02_virsh-capabilities.xml" 2>&1

echo "[3/10] VM별 AppArmor 프로파일..."
ls -la /etc/apparmor.d/libvirt/libvirt-* > "$OUTPUT_DIR/E03_vm-profiles.txt" 2>&1

echo "[4/10] QEMU 프로세스 레이블..."
ps -eZ | grep qemu-system > "$OUTPUT_DIR/E04_qemu-labels.txt" 2>&1

echo "[5/10] AppArmor DENIED 로그..."
grep "apparmor=\"DENIED\"" /var/log/syslog > "$OUTPUT_DIR/E05_denied-logs.txt" 2>&1 || true

echo "[6/10] AIDE 베이스라인..."
stat /var/lib/aide/aide.db > "$OUTPUT_DIR/E06_aide-baseline-info.txt" 2>&1
ls -la /var/lib/aide/aide.db.*.bak >> "$OUTPUT_DIR/E06_aide-baseline-info.txt" 2>&1 || true

echo "[7/10] AIDE 검사 결과..."
ls -la /var/log/aide/ > "$OUTPUT_DIR/E07_aide-checks.txt" 2>&1
tail -100 /var/log/aide/aide-check-$(date +%Y%m%d).log >> "$OUTPUT_DIR/E07_aide-checks.txt" 2>&1 || true

echo "[8/10] 패키지 인벤토리..."
dpkg --get-selections > "$OUTPUT_DIR/E08_dpkg-selections.txt" 2>&1
dpkg -l | grep '^ii' | wc -l > "$OUTPUT_DIR/E08_package-count.txt" 2>&1

echo "[9/10] 네트워크 ACL..."
iptables -L -n -v > "$OUTPUT_DIR/E09_iptables.txt" 2>&1 || true
nft list ruleset > "$OUTPUT_DIR/E09_nftables.txt" 2>&1 || true
ip addr show > "$OUTPUT_DIR/E09_ip-addr.txt" 2>&1

echo "[10/10] 시스템 정보..."
uname -a > "$OUTPUT_DIR/E00_system-info.txt"
lsb_release -a >> "$OUTPUT_DIR/E00_system-info.txt" 2>&1

# 압축
cd "$(dirname $OUTPUT_DIR)" && tar czf "${HOST}_${TIMESTAMP}.tar.gz" "$(basename $OUTPUT_DIR)"
echo "완료: ${HOST}_${TIMESTAMP}.tar.gz"
```

### 13-1. 실행 방법

## 14. 심사 현장 실시간 시연 스크립트

### 14-1. 3분 시연 시나리오

**준비**: 노트북 + 컴퓨트 노드 SSH (Bastion 경유)

| 단계 | 명령 | 설명 |
| --- | --- | --- |
| 1 (30초) | `sudo aa-status` | "85개 프로파일이 enforce 모드로 동작 중" |
| 2 (30초) | `virsh capabilities | grep -A2 secmodel` | "libvirt가 AppArmor로 sVirt 활성 상태" |
| 3 (30초) | `ps -eZ | grep qemu-system | head -3` | "각 VM의 QEMU가 고유 프로파일로 격리 중" |
| 4 (30초) | `cat /etc/apparmor.d/libvirt/libvirt-<uuid>` | "프로파일 내용에 해당 VM만의 경로 명시" |
| 5 (30초) | `sudo ausearch ... / aide --check` | "FIM이 실시간 무결성 감시 중" |
| 6 (30초) | `grep DENIED /var/log/syslog` | "위반 시 즉시 차단 + 로그 기록" |

### 14-2. 주의 사항

- 시연 전 `aa-status` 출력이 enforce 모드인 환경 **반드시 준비**
