1. 카드 시스템 통합 모니터링 시스템 구축 및 활용 

시스템 통합 모니터링 대쉬보드 : MONT

수집 로그:
. 시스템 로그(WAS, DB, 어플리케이션, 클라우드 등)
. 비즈니스 로그
구축 환경:
. frontend : 그라파나
. DB : postgreSQL
. 데이터 수집 : Logstash + Spring Boot
모니터링 지표:
. 주요 시스템 TPS 및 응답시간
. 대고객 채널 로그인 건수
. 주요 시스템 CPU
. 주요 비즈니스 지표(센터컷, 회계 관련)
. 주요 시스템 이벤트

Knox Meeting & Brity RPA를 활용하여 언제어디서든 시스템 현황 모니터링 가능한 환경 구축
- Brity RPA와 Knox Meeting 을 활용하여, 모바일로도 통합 대시보드를 열람 가능 → 이슈 발생 시 신속한 장애 대응을 가능케 함.

마이데이터 모니터링 전용 대시보드
- 마이데이터플랫폼(MDP) 업무 모니터링 전용 대시보드
- SCP 배포된 앱 특성 반영하여 K8s 특화 모니터링 구현(워커노드, Pod 등)

결제일 별 안내 메시지 발송 물량 예측 패널
- 예측 모델을 통해 각 결제일 별 메시지 물량을 사전에 예측하고, 주요 기간 동안 예비 인스턴스를 자동으로 가동하여 시스템 안정성을 제고.

대고객 메시지 발송 현황 패널
- 고객에게 대량 발송되는 메시지의 주요 정보를 확인
- 특정 메시지가 시스템 부하 유발 시, 해당 메시지 담당자와 협의하여 신속하게 발송물량 조정 → 빠른 대응 가능

민생회복 소비쿠폰 실시간 신청현황 대시보드 구성
- 민생회복 소비쿠폰 1/2차 신청기간 중 업무유관자들이 현황을 한눈에 확인할 수 있도록 대시보드 구성
- 채널 별 신청불가 현황을 집계하여, 신청 간 문제 발생 시 빠른 대응 가능

시스템 유입 - 과거 이력 비교 대시보드 구성
- 과거 특정일과 시스템 유입건수 비교하여 패턴을 한눈에 비교할 수 있도록 대쉬보드 구성

보안(디도스) 모니터링 대시보드 구성
- 디도스 공격으로 인해 홈페이지 시스템 이상유무 확인 할 수 있는 대쉬보드 구성
- 서버 CPU 사용율, 로그인 성공/실패 건수 그래프 하나에 표현함

블루원 시스템에 부하를 주는 비지니스 모니터링 대시보드 구성
- 센터컷/배치리소스 모니터링 대쉬보드 구성

대외 연계 시스템 장애 인지를 위한 모니터링 대쉬보드 구성
- 대외 거래 오류건 분석을 통해 장애 기관 인지를 빠르게 할 수 있음

과거 시스템 부하 히스토로 대시보드 구성
- 시스템별, 일시별, 시간별 TPS 및 응답시간 조회 가능

(클라우드) 도메인 별 DB Connection Pool 점유량 모니터링 대시보드 구성
- 부하 상황 Scale-out 시, DB connection pool size 장애 발생 소지
- 도메인 별로 점유중인 pool size 모니터링 통해 장애 소지 제거

유량 제어 시스템(Netfunnel) 실시간 상황 모니터링 대시보드 구성
- Netfunnel과 API 연계하여 대고객 채널의 실시간 대기상황 모니터링 기능 구현

3. 카봇 탄생과 성장 그리고 미래
카봇: 카드IT그룹 챗봇의 줄임말

1. 카봇의 탄생
소통 창구
- 쉽고 빠르게 언제든지 공지도 전달하고, 간단한 업무는 즉시 처리하게 해주는 소통의 도구
이름의 비밀
- 챗봇의 설문 기능(이름짓기 이벤트)를 이용하여 전부서원 참여로 탄생한 카봇 (2023.04.04 탄생)
카봇 초기 활동
- 업무 담당자, 각종 전결기준, 업무관련 URL 정보, 업무 관련 각종 궁금증 FAQ 정보 -> 메신저 통해 빠르게 확인
- 조직활성화 행사, 휴게공간 아이디어, 연구회 가입 -> 설문 기능 이용하여 손쉽게 취합
- 세미나 공지, 상시 시상 안내 -> 공지사항 수시 전달
카봇 사용 후기
- 카봇 오픈후 긍정적 반응과 추가적으로 업무에 활용 가능한 기능개선 추가 요청

2. 카봇의 성장
고된 IT 운영 업무
- 퇴근이후, 공휴일, 외부에서 조치가 가능하면 얼마나 좋을까?
- 카봇을 이용해보자 -> 모바일에서 가능하다는 장점을 살려보자.
카봇으로 적용한 업무 사례
- CD망 예금주 서비스 은행 장애시 타은행으로 전환하여 처리
- 1원 입금 서비스 은행 장애 시 타은행으로 1원 입금 서비스 은행을 전환하여 처리
- FDS 관게항목을 선택하여 모니터링하고 항목별 가동 및 중단 상태를 변경 처리
- 기관별 조회기간을 선택하여 승인된 건수와 전주건수, 전월건수를 비교
- 조회기간내 간편결제 온/오프라인 결제건수와 금액을 모니터링
업무 활용 사례 지속 발굴
- 하반기 적용 계획: 총 36건 예정
- 긴급대응: 대외기관 인터페이스 장애발생시 인터페이스 우회 처리, 실시간 지급 그룹별 처리은행 조회/변경 등
- 모니터링: RestAPI 결재연계 에러건 조회, DB 시스템 현황 조회, 입금한도반영 상태코드별 현황 등
최근 비상연락망도 추가
- 시청역사고로 인해 늦은시간 부서원의 신속한 안전확인이 필요한 상황 -> 카봇에 추가

3. 카봇의 미래
FabriX를 이용한 PoC 기능 추가 (FabriX: 삼성SDS 내부 LLM 서비스)
- 상반기 수행 PoC '인시던트 분석 및 대응' 기능 추가
- 카봇을 이용한 인시던트 분석 및 대응 업무 적용 시나리오
  . 시스템 메시지 발생 -> 담당자 수신 -> 모바일 카봇 'AI 장애분석' 프롬프트 입력 -> 내부데이터 활용한 검색 -> 요약/결과확인(Text)
GenAI 업무 자동화를 위한 Use Case 발굴
- 카드 사내 보안/권한 업무 HAP 활용 자동화 적용 시나리오
  . 인쇄출력 워터마크 예외신청 등

8. 카봇 활용 시스템 이벤트 발생시 즉시 인지 및 대응체계로 장애예방 

매일 자동으로 Knox Meeting 개설 및 시스템 모니터링 내용을 메일로 작성하여 공유
- 시스템 이벤트가 발생하면 개설되어있는 방을 통해 현 상황을 판단하고 조치 여부 결정
- 모바일을 이용하여 장소에 상관없이 모니터링 및 즉시 판단을 통한 조기 대응
  . 회의 등으로 부재중인 경우 자리로 돌아와서 시스템 상황 체크 불필요
  . 자리로 돌아오거나 VDI를 통한 시스템 접속에 필요한 공수 절감
- 적용효과: 회의 등으로 업무시간중 부재중이거나 퇴근 후 비업무 시간 중시스템 이벤트 발생 시 즉각적인 상황판단을 통해 장애 예방 가능

Knox Bot에 사전에 등록해둔 명령 전송(대화 방식)을 통해 모니터링 내용 캡춰한 내용을 메일로 전송받아 빠르게 상황에 대한 세부적인 내용 추가확인 가능

9. 적용 후기: 모니터링 및 긴급대응에 카봇 활용 사례

1. FDS 관제 항목 모니터링(2/29 적용)
피싱사고전송, 피싱탐지 자동보호조치, 피싱탐지 유의대상 자동등록, eFDS 추가인증 전송여부, 출입국 정보 조회 등 현재상태 조회, 가동, 중단 실행 -> 처리성 업무로 카봇에서 컨펌 기능 구현
6월 적용
적용후 실효과: 
이벤트 발생시 대응시간 단축
- 여신협회 장애 또는 시스템 작업시 출입국정보 조회기능을 ON/OFF 처리
  : 업무시간외 여신협회 정기/비정기 작업으로 인한 대응조치 시간 대폭 단축됨
    VDI 접속하기 위해 노트북 상시 휴대 불필요해졌으며, VDI 접속 오류 발생하여 고객센터에 연락하여 처리해야할때 발생하는 접속지연(20분이상)도 우려할 필요 없어짐. 정상적인 VDI접속상황에 비해서도 10분이상 대응속도 향상됨
- eFDS 시스템 응답처리 지연시 추가인증정보 전송 ON/OFF 처리

2. 승인시스템 간편결제 모니터링(24.07 적용)
- 승인시스템 간편결제 모니터링 (24'.07 적용)
  : 2024/10/15 18:20경 SCP 스위치 증설 작업 장애로 온라인 간편결제 영향도 즉시 조회 및 상황파악 
  : 장애영향을 확인 후 VDI 접속 → 상세 영향도 파악 완료 
- VDI접속을 통한 온라인 간편결제 상세영향도 KNOX메일링 기능 개발중 (24'.10 완료예정)
  : VDI 접속 없이 간편결제별 상세영향도 즉시 파악 가능 (분당 건수 및 영향금액 발췌) 
적용후 실효과: 
- Pay 거래가 증가함에 따라 네트워크 장애시 승인거래 영향을 주는 상황이 자주 발생함
  : 온라인 간편결제 영향도에 대한 즉시 파악 니즈가 자주 발생
   최근 DDOS, SCP 증설 장애시 승인 영향도 개략적으로 파악하는 용로로 사용 함
   활용도를 높이기 위해 별도 DB 접속하여 영향도 데이터 발췌가 필요 없도록 완전히 
   카봇으로 영향도 파악을 하는 방향으로 개선 진행 중으로 향후 활용도를 더 높일 수 
   있을 것으로 기대 함

- 주말/퇴근 후 불필요한 VDI 접속 빈도를 줄이고 신속한 영향도 파악 및 상황보고 가능

7. 청구서 마감파일 정리 (Disk 사용량 임계치 도달 시 파일 삭제처리)
pcorbt 서버 내 파일 삭제
 - 마감배치일 하루 최대 20%이상의 디렉토리 사용
 - 배치작업이 남아있는 경우 disk full 되기 전에 용량을 확보해야하므로 vdi로 긴급대응이 필요
 - 파일 삭제 script를 날짜 파라미터로 호출하여 파일 삭제
적용후 실효과: 
- VDI 접근 없이 모바일로 SAM파일 삭제 가능.
- Disk Full로 인한 장애예방효과
- Disk 사용량 임계치 초과/ storage full 상황에서 조치시간 약 0.5~1시간 단축

10. 개인카드 디지털완결 적격대상 중 인적심사 배분 상세사유 조회를 모바일 채널 활용
개인카드 디지털완결 프로세스는 24시간 365일 처리되고 있는데 최근, 휴일동안 발생된 다량의 디지털완결 실패(인적심사 배분처리) 사례가 있었는데 이러한 경우, 디지털완결 실패 건에 대한 조치를 위해 그에 대한 상세원인 분석이 즉시 필요함
적용후 실효과: 
- 개인카드 신청(디지털완결) 과정 중 이벤트 발생시 즉각적인 인지,
- 원인파악도 가능하여 신속한 인시던트 조치 대응

11. 청구서아카이빙 마감작업 완료여부 확인
 - 대량결제일인경우 마감작업이 밤12시 너머로 지연되기도 함
 - 마감 정상여부를 익일 출근하여 확인해야하는데, 재작업시 많은 시간이 소요
 - 마감건수 SQL 호출하여 P_LOG 테이블과 J_LOG 테이블 건수 비교하여 작업마감여부 확인
적용후 실효과: 
- 퇴근후 작업결과 모니터 가능 (기존 익일 확인가능)

12. 대외기관 인터페이스 장애 발생 시, 인터페이스 우회처리 채널로 활용 
카드 신청 시, 입회가부 판정 및 제휴사정보 활용을 위한 대외 인터페이스가 발생하고 있음. 
이중, 대외 정보 미수신일 경우, 후속 업무처리가 안되는 경우가 있음.
(예 : 대외신용기관(NICE, KCB 등)으로 부터 수신되는 정보가 미수신일 경우, 심사진행(디지털완결 포함)이 불가)
따라서 이러한 장애 상황을 대비하여, 
해당 인터페이스가 불가인 경우에는 해당정보를 미수신한 채로 업무처리가 가능하도록 
해당 인터페이스를 OFF 하는 기능을 적용 중인데 현재 블루원 화면을 통한 제어만 가능
야간, 휴일 이벤트 발생 시 모바일을 통해 해당 조치를 하고자 함
적용후 실효과: 
- 예상치 못한 외부환경의 영향(KCB, NICE 등의 인터페이스 장애)으로 문제발생시, 해당 문제해결에 대한 즉각적인 대응이 가능해짐(이전에는 시스템 접속후 블루원 화면을 통해서만 대응 가능 했었음)
