STEM (삼성화재 통합 모니터링 시스템, Samsungfire Transaction End to End Monitoring System)

삼성화재는 손해보험업 국내 1위 금융업계 선두주자
대규모: 9개 보험비즈니스영역, 30만 삼성화재 RC 이용자수, 12682억 영업이익
복잡성: UI/UX, 인터페이스, API 서비스, 서버/인프라/DB, 대외계/보안
관리역량: 25년10월기준 3486대 서버장비수, 645TB 보관데이터, 245462개 트랜잭션수. 수많은 관리 포인트를 제어하기 위한 운영노하우와 역량필요
-> 보험업의 높은 이해도와 숙련된 운영조직인력만이 고객만족과 시스템안정성을 보장할 수 있다.

고객의 질문: 이게 얼마나 도움이 되나요?
Q1: 자동이체 청구서비스가 1000만건이 넘는다면 시스템이 잘 처리할 수 있을까?
-> 대량작업에 대한 가용성 판단 필요
Q2: 입금마감작업이 30초 이상 걸리다보니 컴플레인이 많은데 도대체 왜 늦는걸까?
-> 성능이슈에 대한 신속한 followup 필요
Q3: 월말 대량 납입으로 사용량이 폭증할것으로 예상되는데 어떤 대비를 해야할까?
-> 실시간 Biz 이벤트에 따른 대비책 강구

A3: 월마감일 시스템 안정화 Good! 운영안정성 확보를 위한 근본적 대안 제시

목적과 기능
비즈니스 서비스 시스템 지표 연관성은?
-> 단순기술지표를 보여주는것을 넘어 비즈니스와 시스템 요소를 연결하여 업무영향도를 명확하게 제시
Why STEM?
삼성화재 데이터에 대한 깊은 이해도 & 운영노하우
1단계: 다양한 시스템 인프라 지표, 핵심 데이터 선별 및 수집
2단계: 애플리케이션 트랜잭션 상세 분석
3단계: 비즈니스 이벤트와 시스템 상태 맵핑, 주요서비스에 대한 영향도 상세 분석, 요소별 상호연관성에 대한 정밀 cross-check
4단계: 시스템 종합 모니터링 대시보드 구축, 핵심성과 지표 발굴

삼성화재 고객& 운영자 상호협업
삼성화재의 다양한 고객요구사항
- 비즈니스 이벤트 관리
- 데이터기반 비즈니스 전략 수립
SDS의 운영핵심기술인력
- 시스템 안정성 확보
- 서비스 신뢰성 획득

고객 니즈 실현과 실시간 데이터 수집 분석으로 시스템 운영 안정성 확보 (고객니즈실현+통합모니터링+데이터수집분석+데이터시각화)

고객니즈실현
(대표적인 고객 니즈: 서비스가 느린것 같아요, 시스템은 정상인가요? 문제가 있나요? 이번 월마감은 잘될까요?)
-> 비즈니스 서비스의 신뢰성 확보, 요구사항의 즉각적인 피드백 필요
+
통합모니터링
(통합대시보드 구현: 복잡한 코어시스템 환경, 다양한 지표항목과 임계치, 잠재적 문제에 대한 조기 감지, 쉬운 접근성과 효율적인 관리)
-> 다양한 데이터를 한눈에 파악, 시스템 운영 안정성 향상 기대
+
데이터 수집 분석
(중요 데이터 관리: 실시간 자동화 작업 기반, 핵심지표(KPI)와 시스템 데이터, 고객(사용자) 경험, 필요한 데이터로 빠르게 접근)
-> 중요 데이터 발굴 및 심층 분석, 안정적 운영경험 노하우 반영
+
데이터 시각화
(직관적 표현: 쉽고 빠르게 정보를 이해, 서비스에 대한 직관적인 통찰, 사용자 접근 편의성, 데이터간 상호 연관성 제공)
-> 서비스/시스템 상태 즉시 파악, 올바른 통찰로 이해 공감대 형성

STEM 기술적 구성 (ELK Stack + Grafana 구성환경)

|Grafana|유저 친화적 시각화 도구로 활용|- 고객 경험과 운영노하우를 기반한 맞춤형 View 제공<br>- 데이터 표현시, 직관적인 이해화 통찰중심|

1. 웹/미들웨어 로그
- 웹서버 Access Log 데이터: I/F 수행건수, 서비스 유입정보 등
- 서비스 Exeption 상세 로그: Exception 건수 및 stack 정보 등
2. 애플리케이션
- 서비스 트랜잭션 데이터: 업무별 서비스 발생건수, 응답속도 등
- 비즈니스 포커싱 데이터: 설계 계약건수, 자동이체 청구건수 등
3. 데이터베이스
- DB2 성능지표, 사용량 데이터: Session, GPFS, Latch, Lock Wait 등
- 테이블 버퍼풀, 수행 SQL 상세: Hit Ratio, Read Time, SQL 성능 등
4. 시스템 자원/인프라
- 시스템 성능지표, 인프라 데이터: CPU, 메모리, Threads, DS, Heap 등

=> 시스템 상태 즉각 인지와 선제적 장애 예방 달성
활용1: 실시간 시스템 상태 및 현상 즉각인지 -> 효과1: 정확하고 빠른 판단으로 운영안전성 확보
활용2: 신속하고 정확한 근본원인 분석 -> 효과2: 장애 예측 및 선제적 대응 기반 마련
활용3: 성능 추세 분석 및 패턴 확인 -> 효과3: 비즈니스 전략 및 결정 효율성 증대

고객맞춤 중요 데이터 제공 -> 비즈니스 전략& 판단 근거 자료 활용
1. 삼성화재 설계/계약건수 화면
- 보종별(장기/자동차) 시계열 데이터를 차트로 표현
- 피크타임/일간/월간 사용량 패턴 분석
- 사용량 패턴에 대한 비즈니스 통찰력 제공
- 특정 비즈니스 이벤트(예: 월마감일)에 대한 선제적 대비 가능

2. 삼성화재 모니모 거래현황 화면
- 모니모 관련 시계열 데이터를 차트로 표현
- 전체/개별 서비스의 거래 증감을 시간단위로 제공
- 전체/개별 서비스 사용량 증감에 대한 고객 이해도 상승
- 중요고객(모니모) 서비스에 대한 운영 안정성 확보

3. 삼성화재 트랜잭션 과거 거래일 비교 화면
- 주요 업무별 거래현황 및 평균 응답시간 데이터 제공
- 현재와 과거일(시점) 비교를 통해 불규칙 거래 패턴 확인
- 영향도가 높은 개별 업무 서비스의 불규칙 패턴 조기 감지
- 빠른 고객 담당자 내용 공유로 서비스 안정성 획득

4. 삼성화재 일별 PI I/F 거래 증가패턴 분석 화면 (I/F: Interface)
- 삼성화재 시스템간 I/F 거래건에 대한 상세 데이터 제공
- 특정 I/F 성공률 변화등의 이상패턴 및 이슈 감지
- 비즈니스 이벤트 및 이슈 관련 I/F 서비스 빠르게 감지
- 대고객 I/F 서비스에 대한 시스템 신뢰성 증가

5. 삼성화재 예외발생 현황 상세 화면
- 주요 업무별 서비스 예외발생 상세 데이터 제공
- 오류건에 대한 애플리케이션 담당자의 즉각적인 확인 가능
- 개별 서비스 신뢰성에 대한 KPI 지표로 활용
- 대량 오류건 발생시 시스템 보호장치(유입차단) 역할

중요 데이터 실시간 모니터링 -> 장애 예방 및 운영 안정성 확보

6. 삼성화재 ERP 시스템 상세 모니터링 화면
- 삼성화재 시스템 핵심 정보를 통합된 VIEW로 실시간 제공
- 시스템 이슈 발생 원인 및 영향도 분석에 빠른 접근 가능
- 쉬운 접근성과 직관적인 데이터로 시스템 이해도 상승
- 운영안정성에 대한 공감대 형성(현업, 업무 개발자 등)

7. 삼성화재 BW Context 메모리 화면
- BW 시스템 사용자 수행 작업의 상세 메모리 정보를 표현
- HOST, 커넥션 ID별 최대 메모리 사용 패턴 분석 가능
- 점유 메모리 분석을 통해 OOM 발생 사전 감지
- 높은 사용량 노드 수행되는 SQ에 대한 hint 가이드 권고

8. 삼성화재 BW Expensive SQL 화면
- BW 시스템 사용자 수행작업의 상세 SQL 정보를 표현
- 특정 사용자의 CPU 과점유, 긴 수행시간 SQL을 즉시 확인
- 사용자별 시스템 자원 사용량에 대한 복합적 분석 가능
- 비정상적인 Hang 유발 사용자 SQL의 영향도 검증

9. 삼성화재 BW 메모리 사용현황 화면
- 정보계 시스템 영역별 메모리 사용량 상세 정보를 표현
- 영역별 메모리 사용량 변화 특이점 확인 및 원인 분석
- 특이점 분석을 통한 노드/테이블 상호 연관관계 분석
- 고사용량 노드 분리 및 관련 테이블 대한 파티셔닝 권고

10. 삼성화재 Daily qRFC 모니터링 화면 (qRFC: Queued Remote Function Call, SAP 시스템 간의 통신 매커니즘)
- 삼성화재 ECC/PM 시스템 qRFC 사용량 상세 정보를 표현
- 특정업무 서비스의 사용량 변화를 민감하게 확인
- 큐 처리 지연 및 spike 발생에 대한 상세 분석 자료 활용
- 큐 처리 버그에 대한 SAP OSS 및 Notes 발행 성과 획득
