이창호님 의견은 **기획서의 실현 가능성을 지키는 핵심 지적**입니다. 받아들이는 게 맞고, 받아들이는 방식이 중요합니다.

먼저 두 의견이 충돌하는 지점을 짚겠습니다. 김영도님은 팀장의 "초동이 왜 3일이냐" 모순을 풀려고 **실시간 초동 지원**을 넣었습니다. 이창호님은 그 실시간 연동이 **현재 메가프로젝트가 커버하지 못하는 미지의 범위(2~5번만 가능)**라 빼자고 합니다. 둘 다 맞습니다.

해법은 **실시간을 "기능"에서 빼되, "지표 분리"는 유지**하는 것입니다. 즉 김영도님이 풀려던 모순(초동≠3일)은 실시간 연동 없이도 풀 수 있습니다. 핵심은 이것입니다.

**"3일"의 정체를 다시 정의하면, 실시간 연동 없이도 팀장 모순이 해소됩니다.** 장애 접수 후 데이터가 확보된 상태에서 **분석·교차검증·문서화에 걸리는 리드타임이 3일**입니다. 장애가 3일 터져 있는 게 아니라, *분석 작업 자체*가 3일 걸리는 겁니다. 이 분석 시간을 4시간으로 줄이는 것 — 이게 정확히 메가프로젝트 2~5번이 커버하는 영역입니다. 실시간 연동은 필요 없습니다.

그래서 정리하면: **실시간 초동 지원 기능은 이번 범위에서 제외**하고, 대신 **"분석 리드타임 단축"이 곧 초동·근본원인 양쪽을 모두 빠르게 만든다**는 논리로 팀장 모순을 해소합니다. 범위는 명확해지고 모순도 사라집니다.

아래는 이 방향으로 다시 쓴 전체 기획서입니다.

## 2. 과제 내용

### (1) 추진배경 및 과제내용

**추진배경**

- 장애 접수 후 **데이터가 확보된 상태에서도**, 담당자가 메트릭·로그·이벤트를 개별적으로 수동 분석하여 원인을 도출하고 보고서를 작성하는 데 많은 시간이 소요된다. 이 **분석 리드타임**이 곧 장애 대응 지연으로 이어진다.
- 간헐적 성능 저하·재발성 이슈처럼 데이터 교차분석·재현·문서화가 필요한 건은 분석가 기준 **평균 3영업일의 분석 리드타임**이 발생한다.
- 분석 품질이 **소수 시니어 엔지니어 역량에 종속(속인화)**되어, 인력 부재·야간 대응 시 품질 편차와 지연이 발생한다.
- sar·sosreport·DB 진단 데이터는 **이미 표준화·정형화**되어 있고 수집 자산(PISA·ProbeONE·SAR Analyzer·AWR)도 운영 중이다. → AI가 가장 잘하는 영역(정형 데이터 패턴 인식·요약)과 업무 병목이 정확히 일치하며, **추가 수집 인프라 없이 즉시 효과 검증이 가능**하다.

**과제 범위 — 무엇을 하고, 무엇을 하지 않는가 (Scope Box)**

> **이 Agent가 하는 것:** 장애 접수 후 확보된 데이터의 교차분석 / 이상징후·상관관계 탐지 / 원인 후보 제시(신뢰도 점수 포함) / RCA 초안 자동 생성
> **하지 않는 것:** 데이터 수집기 신규 개발 ✕ / **모니터링 시스템 실시간 연동·자동 장애 감지 ✕** / 자동 원격 조치·명령 실행 ✕ / 최종 원인 확정 ✕ (확정은 엔지니어가 수행 — Human-in-the-loop)

*→ 본 과제는 "장애가 접수되고 데이터가 확보된 상태"를 출발점으로 한다. 모니터링 시스템과의 실시간 연동은 범위·연계 방식이 불확실하여 이번 범위에서 제외하고, 추후 별도 과제로 검토한다. 산출물은 데이터가 아니라 **원인 후보·근거·RCA 초안**이다.*

**추진 목적**

- 확보된 데이터에 대한 **장애 분석 리드타임 단축** (초동 분석·근본 원인 분석 양쪽)
- 운영자 경험 의존형 분석 체계를 **표준화**로 전환
- **RCA 작성 자동화** 및 운영 노하우의 **자산화**
- 반복 장애 대응 효율 향상

**주요 기능**

- **데이터 교차분석:** sar·sosreport·DB 진단 데이터를 전처리·정제 파이프라인으로 대상화한 뒤, AI가 메트릭·로그·이벤트를 교차분석하여 이상징후·상관관계·추정 원인을 신뢰도 점수와 함께 제시한다.
- **분석 표준화:** 담당자 숙련도와 무관하게 일관된 1차 분석 결과를 제공한다.
- **RCA 자동화:** 엔지니어가 검증한 결과를 기반으로 RCA 초안을 자동 생성한다.
- **지식 자산화:** 휘발되던 분석 결과·조치 이력을 신뢰도 점수와 함께 지식베이스에 자동 축적한다.

### (2) 기대효과

> **핵심: 초동 분석과 근본 원인 분석을 분리.** 둘 다 "데이터가 확보된 뒤 분석에 걸리는 시간"을 단축하는 것이며, "3일"은 장애 지속시간이 아니라 **심층 분석의 리드타임**이다.

**정량적 기대효과**

| 구분 | 지표 | 내용 |
|---|---|---|
| **① 초동 분석** | 데이터 분석 작업시간 **수 시간 → 수십 분** | 확보된 성능·로그·이벤트를 AI가 즉시 교차분석하여 원인 후보를 신속 도출 |
| **② 근본 원인 분석(RCA)** | 근본 원인 분석·요약 리드타임 **3일 → 4시간 (83% 단축)** | 간헐 성능저하·재발 이슈·심층 분석 대상 |
| **③ 공수 절감** | 건당 분석 MH **~80% 절감** | 동일 인력으로 처리 가능 분석 건수 증가 → MSP 기술지원 원가 절감 |

**정성적 기대효과**

- **RCA 작성 자동화:** 수작업 → AI 초안 자동 생성
- **운영 지식 자산화:** 개인 경험 의존 → 신뢰도 점수 부여 조직 지식베이스
- **품질 표준화:** 시니어 부재·야간 대응 시에도 일관된 1차 분석 품질 확보

### (3) AI 적용 전/후 업무프로세스

**기존 방식 (As-is)**

① 장애 원인 분석 요청 접수 → ② SAR/SOS Report 수집 → ③ grep·육안으로 메트릭/로그 개별 확인 → ④ 경험 기반 가설 수립 → ⑤ 필요 시 벤더 문의·재현 → ⑥ 원인 확정·RCA 문서 수기 작성

**변경 방식 (To-be)**

① 장애 원인 분석 요청 접수 → ② 데이터 수집·정제 파이프라인 → ③ AI가 전체 데이터를 대상으로 메트릭·로그·이벤트 교차분석 → ④ 이상징후·상관관계·추정 원인을 **신뢰도 점수와 함께** 제시 → ⑤ 엔지니어가 검증·확정 (Human-in-the-loop) → ⑥ RCA 문서 초안 자동 생성

*※ 본 과제 범위는 ②~⑥(데이터가 확보된 이후의 분석·문서화)이다. ①의 장애 감지·접수 및 모니터링 시스템 연동은 기존 운영 체계가 담당한다.*

### (6) 기술 활용계획

**AI 구현 기능 및 적용 Task**

- **데이터 분석:** 수집 데이터 기반으로 메트릭·로그를 통합 분석하여 장애 시점 이상징후 탐지 및 영향 범위 분석
- **원인 후보 제시:** 이상징후·이벤트 상관관계·장애 발생 패턴을 분석하여 발생 가능성 높은 원인 후보를 **신뢰도 점수와 함께** 제시
- **RCA 초안 생성:** 엔지니어가 검증한 결과 기반으로 장애 개요·원인·개선방안·재발방지 대책을 포함한 RCA 초안 자동 생성

**신뢰성 확보 방안(환각 대응):** 모든 분석 결과에 신뢰도 점수를 부여하고, 최종 원인 확정·조치는 반드시 엔지니어가 수행한다(Human-in-the-loop). AI는 자동 조치·명령을 실행하지 않는다.

**최종 산출물 제공 방식:** 자연어 질의 → AI 자동 분석 → 원인 후보·분석 근거·조치 권고·RCA 초안 제공

**필요 기술:** FabriX API(폐쇄망 LLM 추론), Claude Code(개발 생산성), Grafana(분석 결과 시각화)

2. 과제 내용

(1) 추진배경 및 과제내용

추진배경

- 장애 대응 시간의 80% 이상이 데이터 수집 및 분석에 소요되어 실제 원인 판단은 후반부에 이루어짐

- 장애 분석 결과가 엔지니어 역량에 의존하여 분석 품질 편차가 발생함

- 성능·로그 데이터는 표준화되어 있어 AI 기반 분석 적용 효과가 높음

- AI의 패턴 분석 및 요약 역량을 활용하여 장애 대응 효율과 품질을 향상할 필요가 있음

과제내용 상세

추진 목적

- 장애 대응 초기 분석 시간 단축

- 운영자 경험 의존형 분석 체계 개선

- 장애 분석 품질 표준화

- RCA 작성 자동화

- 반복 장애에 대한 대응 효율 향상

- 운영 노하우의 자산화

주요 기능

- AI가 성능·로그 데이터를 1차 분석하고 운영자는 검증 및 의사결정에 집중

- 분석 절차를 표준화하여 담당자의 숙련도와 무관하게 일관된 분석 결과 제공

- 장애 발생 시 원인 후보 및 영향 범위를 실시간 분석하여 초동 대응 지원

- 반복적인 장애 분석 업무를 자동화하여 운영 생산성 향상

- 휘발되던 장애 분석 결과와 조치 이력을 지식베이스에 자동 축적하여 운영 자산화

AI 구현 기능 및 적용 Task

- 장애 원인 후보 제시 : 분석 결과를 기반으로 이상징후, 이벤트 상관관계 및 장애 발생 패턴을 분석하여 발생 가능성이 높은 원인 후보 제시
