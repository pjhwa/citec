## 구성

| 버킷 버전 관리 활성화 여부 | 콘솔에서 버킷 속성 확인 | 정상: 버전 관리 활성화됨<br>취약: 버전 관리 비활성화됨 | 버전 관리가 비활성화되어 객체 덮어쓰기나 삭제 시 이전 버전 복구가 불가능하며, 실수나 악의적 삭제로 데이터 손실 발생 가능함. 데이터 무결성 유지 어려움. | 버킷 속성에서 버전 관리를 활성화하여 객체의 모든 버전을 유지하고, 필요 시 이전 버전으로 복구할 수 있도록 함. | * 설정방법: 콘솔에서 버킷 선택 후 속성 탭에서 버전 관리 활성화 선택.<br>* References: Ceph Documentation - Object Gateway Versioning (https://docs.ceph.com/en/latest/radosgw/versioning/) - Ceph 기반 Object Storage에서 버전 관리 기능 설명, 활성화 이유 및 방법 포함. |
| 버킷 라이프사이클 정책 설정 여부 | 콘솔에서 버킷 관리 섹션에서 라이프사이클 규칙 확인 | 정상: 라이프사이클 정책 설정됨<br>취약: 라이프사이클 정책 없음 | 라이프사이클 정책 미설정 시 오래된 객체가 자동 삭제되지 않아 스토리지 비용 증가하고, 용량 부족으로 성능 저하 발생 가능함. 불필요한 데이터 축적으로 관리 복잡성 높아짐. | 라이프사이클 규칙을 추가하여 객체 생성 후 일정 기간 경과 시 자동 전환 또는 삭제 설정함. 예를 들어, 30일 후 아카이브로 이동. | * 설정방법: 콘솔에서 버킷 관리 > 라이프사이클 > 규칙 추가.<br>* References: Ceph RGW Lifecycle Management (https://docs.ceph.com/en/latest/radosgw/lifecycle/) - Ceph에서 객체 라이프사이클 관리 방법과 비용 절감 효과 설명. |
| 서버 측 암호화(SSE) 활성화 여부 | 콘솔에서 버킷 속성에서 암호화 설정 확인 | 정상: SSE 활성화됨<br>취약: SSE 비활성화됨 | 암호화 미적용 시 데이터 유출 시 민감 정보 노출 위험 높아짐. 규정 준수 위반 발생 가능하며, 보안 취약점으로 공격 대상이 됨. | 버킷 속성에서 SSE를 활성화하고, 기본 키 관리 서비스와 연동하여 모든 객체를 암호화함. | * 설정방법: 콘솔에서 버킷 속성 > 기본 암호화 > SSE 활성화.<br>* References: Ceph RGW Server-Side Encryption (https://docs.ceph.com/en/latest/radosgw/sse/) - Ceph Object Storage에서 서버 측 암호화 설정 및 보안 강화 이유 포함. |
| 버킷 정책을 통한 접근 제어 설정 | 콘솔에서 버킷 정책 섹션 확인 | 정상: 접근 제어 정책 설정됨<br>취약: 정책 없거나 허용 범위 넓음 | 정책 미설정 시 무단 접근 가능하며, 데이터 유출이나 수정 위험 발생함. 공용 버킷처럼 취급되어 보안 사고 유발 가능. | 버킷 정책을 작성하여 특정 IP나 사용자만 접근 허용하도록 세밀하게 설정함. | * 설정방법: 콘솔에서 버킷 정책 편집 > JSON 정책 입력.<br>* References: Ceph RGW Bucket Policies (https://docs.ceph.com/en/latest/radosgw/bucketpolicy/) - Ceph에서 버킷 정책을 사용한 접근 제어 방법과 보안 베스트 프랙티스. |
| ACL(Access Control List) 설정 적절성 | 콘솔에서 객체 또는 버킷 ACL 확인 | 정상: ACL 최소화 사용<br>취약: 공용 ACL 설정됨 | 과도한 ACL 허용 시 권한 남용으로 데이터 노출 발생 가능함. 정책과 중복 시 관리 혼란 초래하며, 보안 홀 생성. | ACL 대신 버킷 정책을 우선 사용하고, 필요 시 세밀한 ACL만 적용함. | * 설정방법: 콘솔에서 객체 속성 > ACL 편집.<br>* References: Ceph RGW ACLs (https://docs.ceph.com/en/latest/radosgw/ACL/) - Ceph Object Storage ACL 설정 가이드와 정책 우선 사용 권고. |
| CORS(Cross-Origin Resource Sharing) 규칙 설정 | 콘솔에서 버킷 CORS 섹션 확인 | 정상: 필요한 CORS 규칙 설정됨<br>취약: CORS 규칙 없거나 과도 허용 | CORS 미설정 시 웹 애플리케이션에서 접근 불가하여 서비스 중단 가능함. 과도 허용 시 CSRF 공격 위험 증가. | CORS 규칙을 추가하여 허용 도메인과 메서드 지정함. | * 설정방법: 콘솔에서 버킷 속성 > CORS > 규칙 추가.<br>* References: Ceph RGW CORS (https://docs.ceph.com/en/latest/radosgw/cors/) - Ceph에서 CORS 구성 방법과 보안 고려사항. |
| 버킷 로깅 활성화 여부 | 콘솔에서 버킷 속성에서 로깅 설정 확인 | 정상: 로깅 활성화됨<br>취약: 로깅 비활성화됨 | 로깅 미적용 시 접근 이력 추적이 불가능하여 보안 감사 어려움. 문제 발생 시 원인 분석 지연됨. | 대상 버킷에 로깅을 활성화하고, 로그 저장 버킷 지정함. | * 설정방법: 콘솔에서 버킷 속성 > 로깅 > 활성화.<br>* References: Ceph RGW Logging (https://docs.ceph.com/en/latest/radosgw/logging/) - Ceph Object Storage 로깅 설정과 감사 목적 설명. |
| 이벤트 알림 설정 여부 | 콘솔에서 버킷 알림 섹션 확인 | 정상: 필요한 이벤트 알림 설정됨<br>취약: 알림 없음 | 알림 미설정 시 객체 변경 시 즉시 대응 불가하며, 자동화 프로세스 지연 발생 가능함. 모니터링 효율 저하. | 이벤트 유형(예: 객체 생성)에 알림을 설정하여 SNS나 Lambda-like 서비스 연동함. | * 설정방법: 콘솔에서 버킷 알림 > 이벤트 추가.<br>* References: Ceph RGW Notifications (https://docs.ceph.com/en/latest/radosgw/notifications/) - Ceph에서 이벤트 알림 구성과 실시간 대응 이점. |
| 버킷 태그링 정책 설정 | 콘솔에서 버킷 태그 확인 | 정상: 태그 적용됨<br>취약: 태그 없음 | 태그 미사용 시 비용 할당과 관리 어려움. 대규모 환경에서 객체 분류 불가하여 운영 효율 저하. | 버킷과 객체에 비용 센터나 환경 태그 적용함. | * 설정방법: 콘솔에서 버킷 속성 > 태그 > 추가.<br>* References: Ceph RGW Tagging (https://docs.ceph.com/en/latest/radosgw/tagging/) - Ceph Object Storage 태그 사용 가이드와 관리 최적화. |
| 객체 잠금(Object Lock) 활성화 여부 | 콘솔에서 버킷 속성에서 객체 잠금 확인 | 정상: 객체 잠금 활성화됨<br>취약: 객체 잠금 비활성화됨 | 잠금 미적용 시 객체가 삭제나 수정될 수 있어 데이터 무결성 훼손 가능함. 규정 준수 요구사항 미달. | 버킷에서 객체 잠금을 활성화하고, 보존 기간 설정함. | * 설정방법: 콘솔에서 버킷 속성 > 객체 잠금 > 활성화.<br>* References: Ceph RGW Object Lock (https://docs.ceph.com/en/latest/radosgw/object-lock/) - Ceph에서 객체 잠금 기능과 데이터 보호 이유. |
| MFA Delete 활성화 여부 | 콘솔에서 버킷 버전 관리 섹션에서 MFA Delete 확인 | 정상: MFA Delete 활성화됨<br>취약: MFA Delete 비활성화됨 | MFA 미적용 시 단일 인증으로 삭제 가능하여 실수나 공격으로 데이터 손실 위험 높아짐. | 버전 관리 활성화된 버킷에서 MFA Delete를 설정함. | * 설정방법: 콘솔에서 버킷 속성 > MFA Delete 활성화.<br>* References: Ceph RGW MFA Delete (https://docs.ceph.com/en/latest/radosgw/mfa-delete/) - Ceph Object Storage MFA 삭제 설정과 보안 강화. |
| 버킷 복제 구성 여부 | 콘솔에서 버킷 복제 섹션 확인 | 정상: 복제 규칙 설정됨<br>취약: 복제 없음 | 복제 미설정 시 지역 장애 시 데이터 가용성 저하됨. | 메타데이터 사용 적절성 | 콘솔에서 객체 속성에서 메타데이터 확인 | 정상: 필요한 메타데이터 적용됨<br>취약: 메타데이터 없음 | 메타데이터 미사용 시 객체 검색과 관리 어려움. 애플리케이션 통합 시 추가 처리 필요하여 성능 저하. | 객체 업로드 시 사용자 정의 메타데이터 추가함. | * 설정방법: 콘솔에서 객체 속성 > 메타데이터 > 추가.<br>* References: Ceph RGW Metadata (https://docs.ceph.com/en/latest/radosgw/metadata/) - Ceph Object Storage 메타데이터 관리와 검색 최적화. |
| 스토리지 클래스 선택 적절성 | 콘솔에서 객체 속성에서 스토리지 클래스 확인 | 정상: 적합한 클래스 선택됨<br>취약: 단일 클래스 사용 | 클래스 미최적화 시 비용 과다 또는 접근 지연 발생함. 자주 접근 객체에 저비용 클래스 사용 시 성능 저하. | 객체 유형에 따라 표준, 아카이브 클래스 선택함. | * 설정방법: 콘솔에서 객체 업로드 시 클래스 선택.<br>* References: Ceph RGW Storage Classes (https://docs.ceph.com/en/latest/radosgw/storage-classes/) - Ceph 스토리지 클래스와 비용-성능 균형 설명. |
