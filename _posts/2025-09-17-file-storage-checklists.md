### 구성 (Configuration)

| 점검내용 | 점검방법 | 점검기준 | 취약시 문제점 | 개선방안 | 참고 |
|----------|----------|----------|---------------|----------|------|
| 볼륨 암호화 활성화 여부 | scpcli filestorage volume show --volume_id <볼륨_ID> | 정상: encryption_enabled = true<br>취약: encryption_enabled = false | 데이터 유출 위험이 증가하며, 규정 준수 위반으로 인한 법적·보안 문제 발생 가능, 복구 시 데이터 무결성 손상 우려 | 볼륨 수정 명령어를 통해 암호화 활성화 설정 적용, 기본 XTS-AES-256 알고리즘 사용으로 데이터 보호 강화 | * 설정방법<br>scpcli filestorage volume set --volume_id <ID> --encryption_enabled true<br>* 확인방법<br>scpcli filestorage volume show --volume_id <ID><br>* References<br>- Overview - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/overview/ (모든 볼륨에 기본 암호화 적용 권장, 데이터 보안 강화) |
| 파일 단위 복구 기능 활성화 여부 | scpcli filestorage volume show --volume_id <볼륨_ID> | 정상: file_unit_recovery_enabled = true<br>취약: file_unit_recovery_enabled = false | 개별 파일 손실 시 전체 볼륨 복구 필요로 다운타임 증가, 운영 효율 저하 및 데이터 복구 시간 지연 발생 | 볼륨 수정 명령어로 파일 단위 복구 활성화, 스냅샷 기반 세밀한 복구 지원 | * 설정방법<br>scpcli filestorage volume set --volume_id <ID> --file_unit_recovery_enabled true<br>* 확인방법<br>scpcli filestorage volume show --volume_id <ID><br>* References<br>- How-to guides - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/how_to_guides/ (파일 단위 복구 기능으로 운영 효율 향상, 활성화 권장) |
| 볼륨 프로토콜 설정 적합성 | scpcli filestorage volume show --volume_id <볼륨_ID> | 정상: protocol = NFS (Linux 환경) 또는 CIFS (Windows 환경)<br>취약: 프로토콜이 연결 서버 OS와 불일치 | 연결 서버에서 마운트 실패로 데이터 접근 불가, 서비스 중단 및 재구성 시간 소요, 호환성 문제로 성능 저하 | 프로토콜 재설정 시 볼륨 재생성 고려, OS 호환 프로토콜 선택으로 안정적 연결 보장 | * 설정방법<br>볼륨 생성 시 --protocol NFS 또는 CIFS 지정<br>* 확인방법<br>scpcli filestorage volume show --volume_id <ID><br>* References<br>- Overview - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/overview/ (NFS: Linux, CIFS: Windows 권장, 호환성으로 안정성 확보) |
| 접근 제어 규칙 등록 여부 | scpcli filestorage volume access rule list --volume_id <볼륨_ID> | 정상: access_rule_state = active, 최소 1개 이상 규칙 존재<br>취약: 규칙 없음 또는 state = inactive | 무단 접근으로 데이터 유출·변조 위험 증가, 보안 취약점 노출로 컴플라이언스 위반 및 감사 실패 | 접근 규칙 추가 명령어로 VM 또는 서버 오브젝트 등록, action = add로 제어 강화 | * 설정방법<br>scpcli filestorage volume access rule set --volume_id <ID> --object_type VM --object_id <ID> --action add<br>* 확인방법<br>scpcli filestorage volume access rule list --volume_id <ID><br>* References<br>- How-to guides - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/how_to_guides/ (접근 제어로 보안 강화, 서버 연결 시 필수 등록) |
| 스냅샷 스케줄 생성 여부 | scpcli filestorage volume snapshot schedule list --volume_id <볼륨_ID> | 정상: 최소 1개 이상 스케줄 존재 (frequency = DAILY 또는 WEEKLY)<br>취약: 스케줄 없음 | 데이터 변경 시 수동 백업 의존으로 복구 지연, 장애 발생 시 데이터 손실 확대 및 RPO(Restore Point Objective) 미달 | 스냅샷 스케줄 생성 명령어로 주기 설정, 자동화로 데이터 보호 일관성 유지 | * 설정방법<br>scpcli filestorage volume snapshot schedule create --volume_id <ID> --hour 23 --frequency WEEKLY --snapshot_retention_count 10<br>* 확인방법<br>scpcli filestorage volume snapshot schedule list --volume_id <ID><br>* References<br>- 스냅샷 사용하기 - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/how_to_guides/snapshot/ (스케줄 자동 생성으로 백업 안정화, 최대 128개 권장) |
| 스냅샷 보관 개수 설정 | scpcli filestorage volume snapshot schedule list --volume_id <볼륨_ID> | 정상: snapshot_retention_count >= 10 (최대 128)<br>취약: retention_count < 5 또는 미설정 | 스냅샷 초과 삭제로 복구 시점 부족, 용량 과다 사용 또는 데이터 손실 위험 증가 | 스케줄 수정 명령어로 보관 개수 조정, 용량 관리로 비용 최적화 및 복구 범위 확대 | * 설정방법<br>scpcli filestorage volume snapshot schedule set --snapshot_schedule_id <ID> --volume_id <볼륨_ID> --snapshot_retention_count 10<br>* 확인방법<br>scpcli filestorage volume snapshot schedule list --volume_id <ID><br>* References<br>- How-to guides - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/how_to_guides/ (보관 개수 1~128 설정으로 용량 조절, 오래된 스냅샷 우선 삭제) |
| 복제 정책 설정 여부 | scpcli filestorage volume replication list --volume_id <볼륨_ID> | 정상: 최소 1개 복제 존재 (replication_type = replication)<br>취약: 복제 없음 | 재해 발생 시 단일 리전 의존으로 서비스 중단, RTO(Recovery Time Objective) 지연 및 비즈니스 연속성 위협 | 복제 생성 명령어로 정책 설정, 다른 리전으로 데이터 보호 강화 | * 설정방법<br>scpcli filestorage volume replication create --volume_id <ID> --replication_type replication --replication_frequency 5min --region kr-west1 --name <이름><br>* 확인방법<br>scpcli filestorage volume replication list --volume_id <ID><br>* References<br>- How-to guides - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/how_to_guides/ (복제 정책으로 DR 강화, 1개 복제본 생성 권장) |
| 복제 주기 설정 적합성 | scpcli filestorage volume replication show --replication_id <ID> --volume_id <볼륨_ID> | 정상: replication_frequency = 5min 이상 (비즈니스 요구에 따라)<br>취약: frequency > 1hour | 데이터 동기화 지연으로 복제본 불일치, 장애 복구 시 데이터 손실 확대 및 복구 시간 증가 | 복제 수정 명령어로 주기 단축, 실시간성 요구에 맞춰 RPO 최적화 | * 설정방법<br>scpcli filestorage volume replication set --replication_id <ID> --volume_id <볼륨_ID> --replication_update_type policy --replication_frequency 5min<br>* 확인방법<br>scpcli filestorage volume replication show --replication_id <ID> --volume_id <ID><br>* References<br>- Overview - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/overview/ (복제 주기 설정으로 데이터 보호, 장애 시 복제 볼륨 활용) |
| 복제 백업 보관 개수 | scpcli filestorage volume replication show --replication_id <ID> --volume_id <볼륨_ID> | 정상: backup_retention_count >= 2<br>취약: retention_count < 2 | 백업 부족으로 복제본 손상 시 복구 옵션 제한, 데이터 영속성 저하 및 재해 복구 실패 위험 | 복제 수정으로 보관 개수 증가, 다중 백업으로 안정성 확보 | * 설정방법<br>scpcli filestorage volume replication set --replication_id <ID> --volume_id <ID> --replication_update_type policy --backup_retention_count 2<br>* 확인방법<br>scpcli filestorage volume replication show --replication_id <ID> --volume_id <ID><br>* References<br>- How-to guides - https://docs.e.samsungsdscloud.com/userguide/storage/file_storage/how_to_guides/ (백업 보관으로 복제 안정화, 최대 2개 권장) |
